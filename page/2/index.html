管理仓库数据和游戏状态（MVC设计模式）

基本设计思路

  • 使用PlayerManager维护玩家状态
  • 使用InventoryManager维护玩家的物品清单
  • 使用Managers来统一管理所有的Manager(如PlayerManager,InventoryManager)
  • 为了让Managers(主管理器)以一种一致的方式来引用其他模块，这些模块必须从一个基类继承共同的属性，将这个共同的基类定义为一个接口

具体代码设计

  • 基本的基类接口(IGameManager)

    //通常以I+接口名的格式来命名接口
public interface IGameManager{
  ManagerStatus status{get;}

  //各个Manager用来初始化的方法
  void Startup();
}

  //管理器所有的可能状态

  • 编写Managers脚本（主管理器）

    [RequireComponent(typeof(PlayerManager))]
[RequireComponent(typeof(InventoryManager))]

public class Managers : MonoBehaviour{
  public static PlayerManager Player{get;private set;}
  public static InventoryManager Inventory{get;private set;}
  //声明

  private List<IGameManager> _startSequence;
  //用于保存Manager的List

  void Awake(){
    Player = GetComponent<PlayerManager>();
    Inventory = GetComponent<InventoryManager>();

    _startSequence = new List<IGameManager>();
    _startSequence.Add(Player);
    _startSequence.Add(Inventory);
    //将Manager添加到List中

    StartCoroutine(StartupManagers());
    //异步启动序列，初始化Manager
  }

  private IEnumerator StartupManagers(){
    foreach(IGameManager manager in _startSequence){
      manager.Startup();
    }

    yield return null;暂停一下

    int numModules = _startSequence.Count;
    //需初始化的模块总数
    int numReady = 0;
    //初始化完毕的模块数

    whilie(numReady<numModules){
      int lastReady = numReady;
      //记录当前初始化完成的模块数
      numReady = 0;

      foreach(IGameManager manager in _startSequence){
        if(manager.status == ManagerStatus.Started){
          //在所有模块初始化完毕之前，如果有模块初始化完毕，则numReady++
          numReady++
        }
      }

      if(numReady>lastReady){
        //如果初始化完毕的模块数大于当前所记录的初始化完毕的模块数（也就是lastReady的值），就更新
        Debug.Log("Process: "+numReady+"/"+numModules);
        yield retrun null;
      }
    }
    Debug.Log("All managers started up");
  }        
}

CRT中电子枪是如何绘图的？什么是场消隐期？

发表于 2018-04-02

CRT显示器中电子枪是如何绘图的？

CRT显像管的成像原理是通过显像管内的电子枪，将光束射出，打在内层玻璃的三原色荧光粉层上，电子束使这些荧光粉层发光，形成图像。
一个像素点由红、绿、蓝三种基色组成，所以要由三个光束打在一个像素点上，并根据光束强弱的不同使红、绿、蓝三种基色的强弱不同，从而显现出各种颜色。由于一个像素点内三种基色靠的很近，人眼无法分辨，所以我们可以在显示器上看到图像。

什么是场消隐期

CRT显像系统通常采用2种扫描方式，即隔行扫描和逐行扫描。
以隔行扫描为例，隔行扫描是，一副图像分两次进行扫描，第一次扫描奇数行的光栅，第二次扫描偶数行的光栅，两场之间的回扫时间称为 场消隐期。

游戏编程算法与技巧习题01

发表于 2018-04-01

1. 为什么早期的游戏使用汇编语言？

早期受家用机性能限制（通常CPU运行在1.1MHz，内存只有128字节），使得用C语言开发不太实际，所以只好用汇编语言开发。

2.什么是中间件？

中间件在操作系统，网络和数据库之上，在应用软件之下，通常是为应用软件提供开发和运行的环境，例如用于开发游戏的Unity和Unreal（虚幻引擎）。并且通过中间件，可以使得应用程序能工作在多平台。

3.选择一款经典的街机游戏，讲一下它们在经典游戏循环中每个阶段都做了什么？

经典的游戏循环为

  • 处理输入
  • 更新游戏世界
  • 生成输出

以拳皇为例：

  • 处理输入：处理摇杆和按键的输入。
  • 更新游戏世界：根据遥感和按键的输入，更新游戏角色的移动，跳跃，转身，以及招式的释放，并且检测招式的命中，以及命中后的效果。
  • 生成输出：更新图像的显示，使玩家可以看到角色移动，受伤等等效果。

4.在传统游戏循环中，有什么渲染之外的输出？

  • 检测器的检测：玩家是否靠近某一物体，触发机关类似的等等
  • 摄像机的更新：玩家控制镜头的移动
    等等

5.一个简单的多线程游戏循环时怎么提升帧率的？

渲染和逻辑更新同步执行，并且让第二条线程（负责渲染的线程）比主线程慢一秒。

6. 什么是输入延迟？多线程游戏循环是怎么导致延迟的？

玩家的输入更久才能在画面上有所反馈。
因为在渲染线程绘制时，不想让主线程简单地只等待渲染结束，所以让渲染线程比主线程慢一秒，从而导致这样的结果
玩家输入->主线程处理输入->一秒后->开始渲染->一段时间后->屏幕上出现（可见）

实现跳跃动作

角色模型时刻受到重力向下拉，当“跳跃”时，应用向上的力

基本思路

  • 进行一系列的判断，来决定角色模型上收到的vertSpeed为多少。

  • 如果角色模型在地面上时，若按下“跳跃键”，则vertSpeed为一个向上的跳跃力，否则vertSpeed为一个很小的向下的力（保证模型在凹凸不平的地面上行走时，被按在地上）。

  • 当角色模型不在地面上时，vertSpeed+=重力乘以一个系数，并且当vertSpeed大于设定的下落的最大速度时，使其等于该速度。从而避免vertSpeed过大

代码（C#）

void Update(){

      vertSpeed = maxFallSpeed;
    }
  }

}

摄像机环绕角色进行旋转

基本实现思路

  1. 获得摄像机和角色模型的初始Transform偏移值
  2. 将按键和鼠标水平移动得到的水平输入值加到摄像机的旋转的y值上，获得旋转后的摄像机的Rotation
  3. 通过角色模型的位置减去旋转后的偏移值，得到摄像机的当前应该处于的位置
  4. 通过代码使摄像机面对模型

代码（C#）：

void Start(){
  _rotY = target.eulerAngles.y;
  _offset = target.position-transform.position;
  //target为角色模型的Transform
}

void LateUpdate(){
  float horInput = Input.GetAxis("Horizontal")
  if(horInput!=0){
    _rotY += horInput*rotSpeed;
  }
  else{
          _rotY += GetAxis("Mouse X")*rotSpeed*3
          //鼠标的水平移动比键盘的更灵敏一些
  }
  Quaternion rotation = Quaternion.Euler(0,_rotY,0);
  //将欧拉角转化为四元数
  transform.position = target.position-(_offset*rotation);
  //Vector3乘以一个四元数得到旋转后的Vector3
  transform.LookAt(target);
}
