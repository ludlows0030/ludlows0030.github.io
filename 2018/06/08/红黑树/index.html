红黑树

红黑树

了解红黑树前先了解2-3查找树

2-3查找树的定义

定义
一棵2-3查找树或为一棵空树，或由以下结点组成：
· 2-结点，含有一个键（及其对应的值）和两条链接，左链接指向2-3树中的键都小于该结点，右链接指向的2-3树中的键都大于该结点
· 3-结点，含有两个键（及其对应的值）和三条链接，左链接指向2-3树中的键都小于该结点，中链接指向的2-3树中的键位于该结点两个键之间，右链接指向的2-3树中的键都大于该结点

2-3树的查找

2-3树的查找与普通二分查找树的类似

向2-结点中插入新键

将2-结点替换为3-结点，将要插入的键保存其中，以保证树的平衡

向一棵只含有一个3-结点的树种插入新键

  • 先临时将新键存入该结点中，使之成为一个4-结点
  • 将一个4-结点转为由3个2-结点组成的树，此时要再细分父结点的情况

向一个父结点为2-结点的3-结点中插入新键

  • 按照前述步骤 将4结点分解后，将位于中间的键移动至原来的父结点中，使之成为一个3-结点

向一个父结点为3-结点的3-结点中插入新键

  • 按照前述步骤 将4结点分解后，将位于中间的键移动至原来的父结点中，使之成为一个4-结点
  • 再继续分解4结点，并将中键向上移动，直至遇见根结点或一个2-结点

分解根结点

  • 当根节点变为4-结点后，将4-结点分解，并为中间的键创造一个新的结点，使得整棵树的高度+1，整棵树仍然保持着平衡

红黑二叉查找树

用于实现2-3查找树的具体数据结构

基本思想

用标准的二叉查找树和一些额外的信息（用来保存3结点）来表示2-3查找树

具体思路

  • 将树中的链接分为两种类型
  • 红链接将两个2-结点连接起来构成一个3-结点
  • 黑链接是2-3树中的普通链接

定义

定义
含有红黑链接并满足下列条件
· 红链接均为左链接；
· 没有任何一个结点同时和两条红链接相连
· 该树是完美黑色平衡的，任意空链接到根结点的路径上的黑链接数量相同

颜色表示

  • 将链接的颜色保存在表示结点的Node数据类型的布尔变量color中
  • 如果指向结点的链接为红，变量为true

  • 反之为false，且约定空链接为false

    特殊情况

    为保证满足 没有任何一个结点同时和两条红链接相连 我们针对3种不同的情况采取措施

  • 结点右链接为红，左链接为黑

  • 结点左链接为红，左结点的左链接依然为红（左结点同两条红链接相连）
  • 结点左右链接都为红

代码（插入算法）

public class RedBlackBST<Key extends Comparable<Key>, Value>
{
    private Node root;

    private static final boolean RED = true;
    private static final boolean BlACK = false;

    private class Node
    {
        Key key;//键
        Value val;//值
        Node left,right;//左右子树
        int N;//结点计数器
        boolean color;//指向该结点的链接的颜色

        Node(Key key,Value val,int N,boolean color)
        {
            this.key = key;
            this.val = val;
            this.N = N;
            this.color = color;
        }
    }

    private boolean isRed(Node x)
    {
        if(x==null) return false;
        return x.color == RED;
    }

    /*
    向左旋转
    将右子树的左子树给父结点
    父结点和右子树交换位置，父结点颜色给右子树，父结点的颜色变为RED
    */
    private Node rotateLeft(Node h)
    {
        Node x = h.left;  
        h.right = x.left;
        x.left = h; 
        x.color = h.color;
        h.color = RED;
        x.N = h.N;
        h.N = size(left)+size(right)+1;
        return x;
    }


    /*
    向右旋转
    将左子树的右子树给父结点
    父结点和左子树交换位置，父结点颜色给左子树，父结点的颜色变为RED
    */
    private Node rotateRight(Node h)
    {
        Node x = h.right;
        h.left = x.right;
        x.right = h;
        x.color = h.color;
        h.color = RED;
        x.N = h.N;
        h.N = size(left)+size(right)+1;
        return x;
    }

    private void filpColor(Node h)
    {
        h.color = RED;
        h.left.color = BlACK;
        h.right.color = BlACK;
    }

    private int size()
    {
        if(x == null) return 0;
        else return x.N;
    }

    public void put(Key key,Value val)
    {
        root = put(root,key,val);
        root.color = BlACK;//保证根结点颜色为BLACK
    }

    private Node put(Node h,Key key,Value val)
    {
        if(h==null) return new Node(key,val,1);//如果当前结点为空，则创建一个新结点
        int cmp = key.compareTo(h.key);
        if(cmp>0) h.right = put(h.right,key,val);//大，找右边
        else if(cmp<0) h.left = put(h.left,key,val);//小，找左边
        else h.val = val;//相等，更新值

        if(isRed(h.right)&&!isRed(h.left)) rotateLeft(h); //右红左不红，向左旋转
        if(isRed(h.left)&&isRed(h.left.left)) rotateRight(h); //左连续红 右转
        if(isRed(h.left)&&isRed(h.left)) filpColor(h); //都红，变色

        h.N = size(left)+size(right)+1;//更新结点计数器
        return h;//返回自身，相当于迭代完成后，沿着路径向上重置
    }

}
